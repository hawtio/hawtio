hawt.io helps you look inside any JVM along with having plugins for working with Apache ActiveMQ, Camel and Karaf.

Click on the tree on the left of the window to browse the various services which are running in this JVM.

As you select different folders and MBeans you will notice the list of tabs at the top left of the screen update to reflect the available features.

You can connect to a different JVM via its Jolokia URL via the Preferences tab

For more help please select the next tabs above.

Click on the org.apache.activemq node in the tree which then exposes the brokers running in a JVM (if there is one)

Within the org.apache.activemq node you can click on the Queue node to see the queues and Topic to see the topics. From either of these nodes you should see a Create Queue or Create Topic tab to be able to create new destinations.

Once you have selected a destination you should be able to Send to it, Browse a queue or view the Attributes or Charts

You can also see a graphical view of all producers, destinations and consumers for all queues (or if you select a Topic folder then topics) using the Diagram tab. Selecting a single queue or topic shows just all the producers and consumers on that destination. This diagram makes it easy to spot if producers are sending messages when there are no consumers, or that consumers are on the wrong destination etc.

Click on the org.apache.camel node in the tree which then exposes any Camel Context running in a JVM (if there is one)

The various child nodes of org.apache.camel lets you view the Attributes or Charts of the various JMX MBeans

You can use the Diagram tab to view a graphical representation of the running routes. If you select a single route instance then the diagram just shows that route only.

Click on the java.lang node in the tree to view numerous JVM level metrics such as OperatingSystem.

The various child nodes of java.lang lets you view the Attributes or Charts of the various JMX MBeans.

Click on the Logs tab to be able to view and search the JVM's logging files (for slf4l and log4j).
